# 2026-07-20 集成三代长读长 eccDNA 分析流程

## 执行摘要

本次更新在 `circdna.nf` 中集成 PacBio / ONT 三代长读长 eccDNA 分析流程：

1. **版本提升**：`v3.0.0` → `v3.1.0`。
2. **参数体系**：新增 `protocol`、`entrypoint`、`long_read_identifier` 等参数，参考 `nanoseq.nf` 与 `isoseq.nf`。
3. **分析引擎**：CReSIL（mapping）+ FLED（mapping）+ Flye（assembly）三引擎。
4. **预处理分支**：PacBio（PBCCS + LIMA）、ONT（CHOPPER + PYCHOPPER）。
5. **模块来源**：nf-core 已有模块直接安装；CReSIL 从 `bio.nf` 复制；FLED 在 `bio.nf` 新建后复制。
6. **结果过滤**：blacklist/repeats 去除 + read support 过滤，统一输出 BED。

## 一、新增参数

### 1.1 `nextflow.config`

```groovy
params {
    protocol              = 'short_read'   // short_read | pacbio | ont
    entrypoint            = 'cleaned_fastq'
    primers               = null
    long_read_identifier  = 'cresil,fled,flye'
    min_read_support      = 2
    blacklist_bed         = null
    repeats_bed           = null
    save_long_read_intermediate = false
    skip_long_read_qc     = false
}
```

### 1.2 `nextflow_schema.json`

- 新增上述参数的 schema 定义。
- 更新 `circle_identifier` help_text，说明长读流程使用 `long_read_identifier`。

### 1.3 `assets/schema_input.json`

- 扩展为支持长读样本表，保留现有 `sample,fastq_1,fastq_2` 兼容。
- 可选增加 `input_bam` 列以支持 PacBio raw subreads 输入。

## 二、新增模块

### 2.1 nf-core 模块（安装到 `modules/nf-core/`）

| 模块 | 用途 |
|------|------|
| `pbccs` | PacBio CCS 纠错生成 HiFi reads |
| `lima` | PacBio 接头/引物切除 |
| `chopper` | ONT 质量/长度过滤 |
| `pychopper` | ONT 接头/引物切除 |
| `flye` | 长读从头组装 |

### 2.2 本地模块（复制/新建到 `modules/local/`）

| 模块 | 来源 | 用途 |
|------|------|------|
| `cresil/trim` | `bio.nf/modules/cresil/trim` | CReSIL 读长修剪 |
| `cresil/identify` | `bio.nf/modules/cresil/identify` | CReSIL eccDNA 鉴定 |
| `fled` | 在 `bio.nf` 分支 `fled` 新建后复制 | 单分子解折叠 eccDNA 检测 |

## 三、新增 Subworkflows

| Subworkflow | 路径 | 功能 |
|-------------|------|------|
| `LONG_READ_PREPROCESSING` | `subworkflows/local/long_read_preprocessing/` | 根据 protocol/entrypoint 完成三代数据清洗 |
| `CRESIL_PIPELINE` | `subworkflows/local/cresil_pipeline/` | CReSIL trim + identify + BED 转换 |
| `FLED_PIPELINE` | `subworkflows/local/fled_pipeline/` | minimap2 比对 + FLED + BED 转换 |
| `FLYE_PIPELINE` | `subworkflows/local/flye_pipeline/` | Flye 组装 + circular 提取 + 回贴 + BED 转换 |
| `LONG_READ_FILTERING` | `subworkflows/local/long_read_filtering/` | 三引擎结果合并、去重、过滤 |

## 四、修改主流程

### 4.1 `workflows/circdna.nf`

- 顶部增加 `protocol` 与 `entrypoint` 校验。
- `protocol == 'short_read'` 时保持现有短读流程不变。
- `protocol == 'pacbio' \| 'ont'` 时：
  1. 跳过 `TRIMGALORE`、`BWA_INDEX`、`BWA_MEM`、短读 `BAM_PREPROCESSING`。
  2. 调用 `LONG_READ_PREPROCESSING`。
  3. 准备参考基因组（fasta、fai、minimap2 index）。
  4. 根据 `long_read_identifier` 调用 `CRESIL_PIPELINE`、`FLED_PIPELINE`、`FLYE_PIPELINE`。
  5. 调用 `LONG_READ_FILTERING`。
  6. 合并 versions 到 MultiQC。

### 4.2 `main.nf`

- 更新 help 信息中的示例命令，补充长读用法。

## 五、配置更新

### 5.1 `conf/modules.config`

- 新增模块的 `publishDir` 与 `ext.args`：
  - `FLYE`：根据 protocol 自动选择 `--pacbio-hifi` / `--nano-hq`。
  - `MINIMAP2_ALIGN`：长读模式 `-x map-pb` / `-x map-ont`，保留 `SA` 标签。
  - `CRESIL_*`、`FLED`：通过 `task.ext.args` 暴露参数。

### 5.2 `conf/base.config`

- 为长读任务分配资源标签：
  - `process_high`：`FLYE`、`CRESIL_IDENTIFY`
  - `process_medium`：`PBCCS`、`LIMA`、`CHOPPER`、`PYCHOPPER`、`FLED`

## 六、测试计划

1. `protocol=short_read` 回归测试：确认二代流程结果不变。
2. `protocol=ont/pacbio, entrypoint=cleaned_fastq, long_read_identifier=cresil,fled,flye` stub 测试。
3. 小数据集验证 CReSIL / FLED / Flye 输出。
4. `nf-core lint` 无致命错误。

## 七、风险与回退

| 风险 | 回退方案 |
|------|----------|
| FLED 模块/容器构建失败 | 第一版先交付 CReSIL + Flye，FLED 后续迭代 |
| CReSIL 资源消耗过大 | 增加资源或仅对小区间运行 |
| 短读流程回归失败 | `protocol` 默认 `short_read`，保持原逻辑隔离 |

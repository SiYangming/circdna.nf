## 二、SRA 转换优化

### 2.1 修改文件

`scripts/convert_sra_to_fastq_parallel.sh`

### 2.2 优化内容

#### 2.2.1 完成标记文件

新增 `sra.completed` 文件，记录已成功转换的 SRA 编号，避免重复转换：

```bash
if [[ -f "$COMPLETED_FILE" ]]; then
    completed_list=$(cat "$COMPLETED_FILE")
    if echo "$completed_list" | grep -q "^$sra_name$"; then
        echo "[SKIP] $species/$sra_name (in completed list)"
        continue
    fi
fi
```

#### 2.2.2 损坏文件检测

新增 `sra.broken` 文件，使用 `vdb-validate` 检查 SRA 文件完整性：

```bash
validate_sra() {
    local sra_file="$1"
    if ! vdb-validate "$sra_file" > /dev/null 2>&1; then
        return 1
    fi
    return 0
}
```

#### 2.2.3 内存限制

将 `--mem` 参数从 100G 调整为 10G，避免内存溢出导致 Segmentation fault：

```bash
--mem 10G
```

#### 2.2.4 降低并发数

将 `MAX_PARALLEL` 从默认值降低为 2：

```bash
MAX_PARALLEL=2
```

#### 2.2.5 终极保底函数

新增 `run_fastq_dump_ultimate` 函数，作为第三重 fallback：

```bash
run_fastq_dump_ultimate() {
    local sra_name="$1"
    local output_sra_dir="$2"
    $SRA_TOOLS_BIN/fastq-dump \
        --split-3 \
        "$sra_name.sra"
    return $?
}
```

#### 2.2.6 删除 sra-tools:3.4.1 代码

移除所有涉及 `SRA_TOOLS_FALLBACK_IMAGE` 和 3.4.1 版本的代码，只保留 3.2.1 版本。

#### 2.2.7 改为本地运行

修改脚本为直接调用本地 `$SRA_TOOLS_BIN` 中的工具，不再使用 Docker。

#### 2.2.8 添加 copy_existing_fastq 函数

支持直接复制已存在的 FASTQ 文件到目标目录：

```bash
copy_existing_fastq() {
    local species="$1"
    local source_dir="$2"
    local output_dir="$3"
    # 查找并复制 .fq.gz 和 .fastq.gz 文件
}
```

### 2.3 修复的问题

| 问题 | 原因 | 修复方案 |
|------|------|---------|
| Segmentation fault | fasterq-dump 内存设置过高（100G），多进程并发导致内存溢出 | 将 --mem 调整为 10G |
| SRA 文件损坏导致转换失败 | 无文件完整性检查 | 添加 vdb-validate 验证 |
| 重复转换 | 无完成记录 | 添加 sra.completed 文件 |
| SRA_TOOLS_BIN 路径错误 | 路径配置错误 | 修正为 /data1/users/siyangming/biosoft/sratoolkit.3.2.0-centos_linux64/bin |

## 四、--genome 参数支持

### 4.1 修改文件

`workflows/circdna.nf`

### 4.2 修改内容

原代码直接检查 `params.fasta`：

```groovy
if (params.fasta) { ch_fasta = channel.fromPath(params.fasta) } else { exit 1, 'Fasta reference genome not specified!' }
```

修改后支持通过 `--genome` 参数获取：

```groovy
if (params.fasta) { 
    ch_fasta = channel.fromPath(params.fasta) 
} else {
    def genome_fasta = WorkflowMain.getGenomeAttribute(params, 'fasta')
    if (genome_fasta) {
        params.fasta = genome_fasta
        ch_fasta = channel.fromPath(genome_fasta)
    } else {
        exit 1, 'Fasta reference genome not specified!' 
    }
}
```

### 4.3 工作原理

1. 如果直接指定了 `--fasta` 参数，使用它
2. 如果没有指定 `--fasta`，尝试通过 `WorkflowMain.getGenomeAttribute(params, 'fasta')` 从 `params.genomes` 配置中获取
3. 如果都没有，报错退出
